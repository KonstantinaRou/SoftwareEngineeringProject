Μη λειτουργικές απαιτήσεις

1.Το σύστημα θα πρέπει να έχει γρήγορη απόκριση σε περίπτωση αστοχίας. (Διαθεσιμότητα)
2.Η ταχύτητα απόκρισης του συστήματος θα πρέπει να είναι υψηλή ώστε να μπορεί να αυξηθεί η αποδοτικότητα.(Απόδοση)
3.Το σύστημα θα πρέπει να προστατεύει τους χρήστες από πιθανή έκθεση των στοιχείων τους σε μη εξουσιοδοτημένους χρήστες.(Ακεραιότητα)
4.Το λογισμικό θα πρέπει να προσφέρει την κατάλληλη συντηρησιμότητα ώστε να μπορεί να τροποποιηθεί και να διορθωθούν τυχόν σφάλματα με ευκολία. (συντηρησιμότητα)
5.Το σύστημα θα είναι αξιόπιστο ως προς την λειτουργία του και την διάρκεια στο χρόνο. (Αξιοπιστία)
6.Το σύστημα θα μπορεί να ανακάμπτει μετά από πιθανή εσφαλμένη χρήση. (Ευρωστία)
7.Το λογισμικό του συστήματος θα είναι εύκολο προς εντοπισμό σφαλμάτων.(Ελεγξιμότητα)
8.Το σύστημα θα πρέπει να είναι εύχρηστο, εύκολο στην εκμάθηση του και φιλικό προς τον χρήστη.(Ευχρηστία)
9.Το σύστημα θα μπορεί να χρησιμοποιηθεί από χρήστες χωρίς ιδιαίτερες γνώσεις χρήσης ηλεκτρονικού υπολογιστή.(Ευχρηστία)
